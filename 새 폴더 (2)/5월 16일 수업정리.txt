기본 데이터 ● 논리 타입
기본 데이터 타입은 클래스를 형성하는 주위 껍질이 없고, 오직 해당 값의 비트 패턴을 저장하기 위한 공간만을 즉시 할당받는다.

c언어와 비교하여 boolean, byte 타입이 추가되었고, 모든 정수타입은 signed로 고정되며, char는 2바이트 자바 유니코드 저장을 위해 unsigned로 고정된다.

축소 대입 시, 반드시 타입 변환 명시를 해주어야 한다.

참/거짓이 필요한 곳은 반드시 boolean 타입을 사용해야 한다.


기본 데이터 ● 논리 ● 문자열 포장 클래스
기본 타입 클래스는 기본 타입에 클래스라는 껍질을 덧씌우기 때문에 포장 클래스라 부른다.

기본 타입 클래스에, c언어에는 존재하지 않는 string 타입이 포함된다.

기본 타입 클래스는 '=' 연산자나 new 연산자중 어느쪽으로도 생성 가능하다.
포장 클래스 객체를 new 연산자로 생성할 때는 각 객체의 생성자가 초기 값에 대하여 엄격한 타입 매칭을 요구한다.

서로 다른 기본 타입 클래스는, 클래스 관점에서 상속 관계가 아니어서 완전히 다르기 대문에 서로 간에'='연산자에 의한 대입이나 수식 연산,다형성의 적용이
불가능하다.


포장 클래스의 비교 연산
자바의 객체 생성 문 "ABC obj = new ABC ()"의 의미는 클래스 ABC를 위한 객체 저장 공간을 확보하고 그 위치를 변수 obj에 저장한다.

일반적으로 "if(obj1 == obj2)..."와 같이 객체 지시자를 비교하면 객체의 저장내용이 아닌 위치를 비교하게 된다.

값이 동일한 객체를 자동포장으로 여러번 생성하면 여러 개의 지시자들이 하나의 객체를 공유한다.


포장 클래스 내용(값) 접근 메소드
포장 클래스의 저장 값은 Value(), toString(), equals() 등의 메소드로 접근할 수 있다.

한번 생성된 포장 클래스의 내용값은 수정할 수 없다.(공유가 가능한 이유)

equals()의 경우 비교 대상이 해당 타입과 동일한 경우에만 정확한 결과를 얻을 수 있다.


열거 타입('enum')
열거형 enum은 임의의 별명을 상수 코드를 대신하여 식별용으로 사용할 수 있도록 지원한다.

별명들의 묶음에 이름을 부여하여 하나의 사용자 정의 기본타입으로 정의한다.

해당 타입의 변수는 묶음 내에 열거된 별명만을 값으로 가질 수 있다.

묶음 내 별명들에게는 내부적으로 0부터 시작해서 연속되는 정수가 차례로 대응된다.

열거 타입 변수는 대입 연산과 일치 여부 비교 연산만 가능하다.
열거 타입 변수는 대소 비교가 불가능하다.
