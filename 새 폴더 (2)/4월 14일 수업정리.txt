동일 소스 파일 내 클래스간 접근 제한 모드
    동일 파일에 작성된 클래스들은 동일 패키지에 속한다.

    접근 제한 모드 public, protected, private 등이 설정되지 않으면 패키지 내에서 접근 가능한 default 모드로 간주한다.

클래스 별 소스 파일 분리
    소스 파일을 분리하더라도 패키지를 지정하지 않으면 동일 클래스이다.

소스 파일 별 패키지 간 접근 제한 모드
    두 클래스의 패키지가 다르고, 참조되는 클래스에 접근 제한 모드가 비어있으면 default모드가 적용된다.

    따라서 참조되는 클래스는 패키지가 다른 클래스로부터의 참조를 허용하지 않는다.        

참조되는 클래스의 접근 제한 모드 설정
    접근 허용이 필요한 클래스, 생성자, 메소드의 접근 제한 모드를 public으로 설정한다.

    외부로부터의 접근을 불허하는 필드는 private로 설정한다.

    하나의 파일 내에 여러 클래스가 존재할 수 있으나 public 클래스가 있다면 오직 하나만 존재해야 하고, 
    이 경우 소스 파일명은 그 public 클래스의 이름과 반드시 일치해야 한다.

접근 제한 모드의 상속(필드)
    public과  protected는 그대로 상속되지만, private는 상속되지 않고 자식 클래스에서의 접근이 허용되지 않는다.

    private 필드와 메소드는 무조건 해당 클래스에서만 접근이 가능하다.   

    메소드 오버라이딩 : 상속 관계에 있는 부모 클래스에서 이미 정의된 메소드를 자식 클래스에서 같은 시그니쳐를 갖는 메소드로 다시 정의하는 것.
