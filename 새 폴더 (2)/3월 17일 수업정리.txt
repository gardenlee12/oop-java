객체 개념 적용하기
자바에서의 static은 오로지 클래스 멤버 변수나 멤버 함수의 정적 여부 표시 용도로만 사용 가능하다.
static 멤버 변수의 경우 객체와 독립적으로 오직 한곳에 존재하게 된다.    

C언어로 지역변수(자동변수)는 함수의 호출이 끝나면 할당된 메모리공간이 초기화되기때문에
값을 계속 유지할 수 없다.
함수내에선 static을 이용하여 고정변수로 변환할 수 없다.
전역변수로 사용해야하는데, class내에서, 메소드 밖에서 변수를 선언하여 사용해야한다.
->class 멤버변수로 등록하여 객체화 한 후 사용.

static을 붙인 변수나 메소드는 객체화되어 여러번 호출하더라도 한가지만 존재한다.(singleton)

java에선 class 밖이나 class 사이에는 코딩이 불가하다.

class Add_51{
int total=0;
int add2i(int a, int b){
int s;
s= a+b;
total +=s;
return (s);
}
}
위 클래스의 경우 여려번 호출해도 total이 초기화되서 매 호출마다 total의 값이 초기화가 되지만

class Add_52{
static int total;
int add2i(int a,int b){
int s;
s = a+b;
total += s;
return (s);
}
}
위 클래스의경우 total을 static을 사용하여 객체화했기 때문에 호출할때마다 따로 초기화를 해주지 않으면 값이 게속해서 누적된다.

동일 기능 이질 타입 메소드 
클래스의 멤버 하무는 해당 객체에 접근하는 방법을 제공한다는 의미에서 메소드라 부른다.
클래스의 멤버 변수는 필드라 부른다.

작은 타입에서 큰 타입으로 형변환은 묵시적으로 자연스럽게 이루어진다.
char-> short -> char -> int > long -> float -> double (c언어 기준)

반대로 큰 타입에서 작은 타입으로 묵시적 형변환은 허용되지 않기 때문에 값의 손실 왜곡을 감수한다는 프로그래머의 명시적 형변환 지시가 필요.
